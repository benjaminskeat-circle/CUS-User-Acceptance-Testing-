Hosting and database guidance

  1. No database needed for this version: this app runs fully in-browser and exports JSON or CSV files for handoff.
  2. If you want persistence: create a Supabase project with tables for testers, exercises, results, and defects; then replace the in-memory arrays with API calls.
  3. Suggested tables: testers(id,name,access_level), modules(id,name), exercises(id,module_id,title,role_scope), results(id,tester_id,exercise_id,status,notes,executed_at), defects(id,result_id,title,severity,userback_ref,details,status).
  4. Suggested hosting: GitHub Pages for static hosting, or Netlify/Vercel if you later add serverless functions.
  5. Supabase setup: create project, define the tables above, enable Row Level Security, create policies by tester role, then store the project URL and anon key in the app config.
  6. Deployment: rename this file to index.html, push to a GitHub repo, enable Pages, and share the published URL.
